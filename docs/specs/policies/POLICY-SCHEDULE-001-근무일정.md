# POLICY-SCHEDULE-001 근무일정

## 정책 규칙

### 1. 근무 일정 생성
1. SUPER_ADMIN 또는 MANAGER가 근무 일정을 등록할 수 있다.
2. MANAGER는 자신의 소속 매장 근로자에 대해서만 일정을 등록할 수 있다 (에러코드: SCH005).
3. 근무 일정은 날짜(workDate), 시작 시간(startTime), 종료 시간(endTime)으로 구성된다.
4. 동일 근로자에 대해 동일 날짜에 중복 일정을 등록할 수 없다 (에러코드: SCH002).
5. 등록 대상 근로자는 해당 매장에 소속되어 있어야 한다 (에러코드: SCH004).

### 2. 근무 일정 수정
1. SUPER_ADMIN 또는 MANAGER가 근무 일정을 수정할 수 있다.
2. MANAGER는 자신의 소속 매장 근로자의 일정만 수정할 수 있다 (에러코드: SCH005).
3. 확정(CONFIRMED) 상태의 근무 일정은 수정할 수 없다 (에러코드: SCH003).

### 3. 근무 일정 조회
1. EMPLOYEE는 본인의 근무 일정만 조회할 수 있다.
2. MANAGER는 소속 매장의 전체 근로자 일정을 조회할 수 있다.
3. SUPER_ADMIN은 모든 매장의 근무 일정을 조회할 수 있다.
4. 월간/주간 캘린더 뷰를 지원한다.

### 4. 근무 일정 구성 요소
| 필드 | 설명 | 제약 조건 |
|------|------|-----------|
| scheduleId | 일정 고유 식별자 | UUID 형식 |
| employeeId | 근로자 식별자 | 유효한 근로자 ID |
| workDate | 근무 날짜 | LocalDate (YYYY-MM-DD) |
| startTime | 근무 시작 시간 | LocalTime (HH:mm) |
| endTime | 근무 종료 시간 | LocalTime (HH:mm), startTime보다 이후 |

### 5. 근로자 유형별 스케줄 차등 관리
1. 정규직(REGULAR): 주 5일 기본 근무 (예: 09:00~18:00).
2. 계약직(IRREGULAR): 계약 조건에 따른 근무 일정 설정.
3. 아르바이트(PART_TIME): 시간 단위 근무 일정 설정 (유연한 시간대).

### 6. 에러 코드
| 에러코드 | 설명 |
|----------|------|
| SCH001 | 근무 일정을 찾을 수 없음 |
| SCH002 | 동일 근로자의 동일 날짜에 중복 일정 |
| SCH003 | 확정된 일정은 수정 불가 |
| SCH004 | 근로자가 해당 매장에 소속되지 않음 |
| SCH005 | 매니저는 자신의 매장만 관리 가능 |

## 적용 대상
- 근무일정(WorkSchedule) 도메인: 일정 생성/수정/조회
- 출퇴근(Attendance) 도메인: 출퇴근 상태 판정 시 근무 일정의 시작/종료 시간 참조
- 급여(Payroll) 도메인: 근무 유형(평일/야간/주말/공휴일) 판단 시 일정 참조
