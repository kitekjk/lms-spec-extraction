# POLICY-ATTENDANCE-001 출퇴근

## 기본 정보
- type: policy
- id: POLICY-ATTENDANCE-001
- last-updated: 2026-03-09

## 정책 규칙

### 1. 출근 처리
1. 근로자는 `POST /api/attendance/check-in` API로 출근을 기록한다.
2. 출근 시 로그인(JWT 인증) 상태여야 한다 (POLICY-AUTH-001 참조).
3. 동일 날짜에 이미 출근 기록이 있으면 중복 출근이 불가하다 (에러코드: ATT002).
4. 출근 시각은 서버 시간(Asia/Seoul) 기준으로 자동 기록된다.
5. 출근 처리 직후 출퇴근 상태는 PENDING(퇴근 대기 중)으로 설정된다.

### 2. 퇴근 처리
1. 근로자는 `POST /api/attendance/check-out` API로 퇴근을 기록한다.
2. 출근 기록 없이 퇴근할 수 없다 (에러코드: ATT003).
3. 동일 날짜에 이미 퇴근 기록이 있으면 중복 퇴근이 불가하다 (에러코드: ATT004).
4. 퇴근 시각은 서버 시간(Asia/Seoul) 기준으로 자동 기록된다.

### 3. 출퇴근 상태 판정
1. 시스템은 5가지 출퇴근 상태를 정의한다:
   - NORMAL: 정상 출근 (근무 시작 시간 이전 또는 동시에 출근)
   - LATE: 지각 (근무 시작 시간 이후 출근)
   - EARLY_LEAVE: 조퇴 (근무 종료 시간 이전 퇴근)
   - ABSENT: 결근 (출근 기록 없음)
   - PENDING: 퇴근 대기 중 (출근 후 퇴근 전 상태)

### 4. 출퇴근 기록 수정
1. MANAGER는 소속 매장 근로자의 출퇴근 기록을 수정할 수 있다.
2. SUPER_ADMIN은 전체 매장의 출퇴근 기록을 수정할 수 있다.
3. EMPLOYEE는 출퇴근 기록을 수정할 수 없다.
4. 수정 시 adjusted 플래그가 true로 설정된다.
5. 모든 수정 이력은 감사 로그(AuditLog)에 기록된다.

### 5. 출퇴근 기록 조회
1. EMPLOYEE는 본인의 출퇴근 기록만 조회할 수 있다.
2. MANAGER는 소속 매장 근로자의 출퇴근 기록을 조회할 수 있다.
3. SUPER_ADMIN은 전체 매장의 출퇴근 기록을 조회할 수 있다.
4. 날짜별, 상태별 필터링을 지원한다.

### 6. 근무 시간 계산
1. 근무 시간 = 퇴근 시각 - 출근 시각 (시간 단위, 소수점 2자리).
2. 퇴근 기록이 없는 경우(PENDING 상태) 근무 시간을 계산하지 않는다.
3. 계산된 근무 시간은 급여 산정에 사용된다 (POLICY-PAYROLL-001 참조).

### 7. 에러 코드
| 에러코드 | 설명 |
|----------|------|
| ATT001 | 출퇴근 기록을 찾을 수 없음 |
| ATT002 | 이미 출근 처리됨 (동일 날짜 중복 출근) |
| ATT003 | 출근 기록 없음 (퇴근 처리 불가) |
| ATT004 | 이미 퇴근 처리됨 (동일 날짜 중복 퇴근) |

## 적용 대상
- 출퇴근(Attendance) 도메인: AttendanceRecord 엔티티의 생성/수정/조회
- 급여(Payroll) 도메인: 출퇴근 기록 기반 근무 시간 계산 및 급여 산정
- 감사(AuditLog) 도메인: 출퇴근 기록 수정 시 변경 이력 추적
